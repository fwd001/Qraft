Android 扫码模块 —— 产品需求文档

  ---
  一、功能概述
  
  在 Android APP 中实现一个完整的二维码扫描与识别模块，支持摄像头实时扫码、连续扫码、相册图片解析三种方式，将二维码内
  容转换为可读文本，并提供结果操作与历史记录管理。

  ---
  二、功能清单与交互细节
  
  模块 A：摄像头实时扫码

  功能目标： 打开后置摄像头，对准二维码自动识别，识别成功自动停止并展示结果。

  交互流程：

  1. 用户进入扫码页面，页面中央显示取景框（半透明遮罩 + 圆角镂空区域），底部显示"开始扫描"按钮
  2. 首次使用时弹出系统摄像头权限申请弹窗
    - 用户拒绝 → 页面显示"已拒绝摄像头权限，请在系统设置中允许访问"，并提供"去设置"入口
    - 用户同意 → 进入下一步
  3. 用户点击"开始扫描"
    - 后置摄像头启动，画面全屏铺满，取景框居中悬浮
    - 若摄像头启动失败，显示错误提示（如"无法打开摄像头"）
    - 摄像头区域底部显示提示文字"对准二维码即可自动识别"
  4. 用户将二维码对准取景框
    - 每约 200ms 取一帧画面进行分析，帧间不卡顿
    - 识别过程中无任何 UI 变化，保持静默检测
  5. 识别成功
    - 摄像头自动关闭
    - 页面从取景状态切换到结果展示状态
    - 显示识别到的文本内容（等宽字体，保留原始换行/格式）
    - 自动将该内容存入扫描历史
  6. 识别失败（未找到二维码）
    - 持续扫描不中断，不显示任何错误提示
  7. 用户可在扫描过程中点击"停止"按钮手动关闭摄像头

  ---
  模块 B：连续扫码
  
  功能目标： 一次性扫描多个二维码，自动去重并按顺序拼接结果。

  交互流程：

  1. 在开始扫描前，用户可通过 Switch 开关切换"连续扫码"模式（默认关闭）
    - 开关标注文字："连续扫码"
    - 扫描进行中时开关置灰不可操作
  2. 用户打开连续扫码开关，点击"开始连续扫描"
    - 摄像头启动，与单次扫码相同的视觉呈现
    - 底部按钮变为"结束扫码"
  3. 用户对准第一个二维码
    - 识别成功后 Toast 提示"已识别第 1 个二维码"
    - 页面显示提示文字"已识别完成，请切换下一张二维码或点击'结束扫码'"
    - 进入 2 秒冷却期（此期间不会重复识别同一个码，给用户时间换码）
    - 页面实时展示已拼接的完整文本
  4. 用户换到第二个二维码（不同内容）
    - 冷却期过后自动开始检测
    - 识别成功 → Toast "已识别第 2 个二维码"，拼接文本更新
  5. 用户不小心把同一个码又扫了一遍
    - 系统检测到内容重复 → 提示"此码已识别，请换下一张"，不重复记录
  6. 用户点"结束扫码"
    - 摄像头关闭
    - 将所有已识别的文本按顺序拼接为一份完整结果
    - Toast 提示"连续扫码结束，共识别 N 个二维码"
    - 拼接后的完整文本存入扫描历史
  7. 用户还没扫到任何二维码就点了"结束"
    - 显示提示"未识别到任何二维码"

  ---
  模块 C：相册图片解析
  
  功能目标： 从手机相册选择一张图片，识别其中的二维码。

  交互流程：

  1. 扫码页面提供"上传图片解析"入口（按钮或图标）
  2. 用户点击后唤起系统相册选择器，仅展示图片类型文件
  3. 用户选择一张图片
    - 非图片文件不可选（系统相册已过滤）
  4. 选中后页面自动开始解析
    - 解析过程中可显示 Loading 态（简短）
    - 解析成功 → 展示结果，自动存入历史
    - 解析失败（无二维码）→ 提示"未识别到二维码"
    - 图片加载失败 → 提示"图片加载失败"

  ---
  模块 D：结果展示与操作
  
  功能目标： 识别成功后提供清晰的结果查看和便捷操作。

  交互细节：

  1. 结果展示区域
    - 等宽字体展示文本内容，保留原始换行和格式
    - 内容较长时可滚动查看
    - 背景为暗色卡片，文字为亮色高对比度
  2. 复制按钮
    - 点击后文本复制到系统剪贴板
    - 复制成功后 Snackbar / Toast 提示"已复制"
  3. URL 智能识别
    - 系统自动检测结果是否为 HTTP / HTTPS 链接
    - 如果是链接，结果区额外展示"打开链接"按钮
    - 点击后通过系统浏览器打开该链接
  4. 查看详情
    - 提供"查看"按钮，点击后弹出全屏或大尺寸展示模态
    - 模态内可用左右滑动或方向箭头切换连续扫码的多段结果

  ---
  模块 E：扫描历史
  
  功能目标： 记录用户所有扫码结果，方便回溯和复用。

  交互细节：

  1. 自动记录
    - 每次扫码 / 图片解析成功，内容自动存入本地
    - 连续扫码场景：合并后的完整文本才存入历史（单段不存）
  2. 历史列表
    - 按时间倒序排列，最近的在最上方
    - 每条记录展示：
        - 内容摘要（最多 2 行，超出省略）
      - 扫描时间（格式：2025-06-15 14:30）
      - 字符数（如"256 字符"）
    - 最多保留 20 条，旧记录自动淘汰
  3. 点击历史记录项
    - 一键复制该条内容到剪贴板
    - Toast 提示"已复制"
  4. 清空历史
    - 列表顶部提供"清空"按钮
    - 点击后弹出确认弹窗："确定清空所有扫描历史？"
    - 确认后全部删除，列表显示空态"暂无记录"
  5. 数据持久化
    - 使用 APP 本地存储（SharedPreferences / DataStore）
    - APP 卸载后数据丢失（不作云端同步）

  六、测试清单
  
  功能测试（按模块）

  A. 摄像头扫码
  - [ ] 点击"开始扫描"→ 后置摄像头正常启动，画面流畅
  - [ ] 对准有效二维码 → ~0.5s 内识别成功，自动停止，结果正确展示
  - [ ] 对准无效图案 → 不误识别，持续扫描无卡顿
  - [ ] 扫码成功 → 结果自动存入历史
  - [ ] 点击"停止"→ 摄像头正常关闭，资源释放
  - [ ] 拒绝权限后再点击"开始扫描"→ 引导去设置页面
  - [ ] APP 切后台再回前台 → 扫码恢复（或需重新点击开始）
  - [ ] 从扫码页返回到上一页 → 摄像头释放，不消耗电量

  B. 连续扫码
  - [ ] 打开连续扫码开关 → 点击"开始扫描"进入连续模式
  - [ ] 依次扫描 3 个不同二维码 → 每个识别后 Toast 计数正确（第1/2/3个）
  - [ ] 扫描同一个二维码两次 → 第二次提示"此码已识别，请换下一张"
  - [ ] 点击"结束扫码"→ 结果拼接正确，存入历史
  - [ ] 冷却期（2秒）内换码 → 不会立即再次触发识别
  - [ ] 扫码 0 个点"结束"→ 提示"未识别到任何二维码"

  C. 图片解析
  - [ ] 选择 PNG 图片 → 解析成功，内容正确
  - [ ] 选择 JPEG 图片 → 解析成功，内容正确
  - [ ] 选择无二维码的图片 → 提示"未识别到二维码"
  - [ ] 选择损坏的图片文件 → 提示"图片加载失败"

  D. 结果操作
  - [ ] 点击"复制"→ 剪贴板内容正确，Toast 提示"已复制"
  - [ ] 结果为 URL → 显示"打开链接"按钮，点击可跳转浏览器
  - [ ] 结果为普通文本 → 不显示"打开链接"按钮

  E. 历史记录
  - [ ] 扫码后历史列表新增一条记录，排在最上方
  - [ ] 记录显示内容摘要 + 时间 + 字符数
  - [ ] 点击历史项 → 复制内容成功
  - [ ] 点击"清空"→ 确认弹窗 → 确认后列表为空
  - [ ] 历史超过 20 条 → 旧记录自动移除
  - [ ] 关闭 APP 再打开 → 历史记录仍在

  F. 交互细节
  - [ ] Toast 提示约 1.8s 后自动消失
  - [ ] 取景框清晰可见，半透明遮罩 + 圆角镂空
  - [ ] 深色主题全局一致（除相机画面外）
  - [ ] 连续扫码开关在扫描中置灰不可点

  兼容性测试

  - [ ] Android 10 / 11 / 12 / 13 / 14 / 15
  - [ ] 不同厂商：小米 / 华为 / OPPO / vivo / 三星（CameraX 兼容）
  - [ ] 平板设备 / 无摄像头设备
  - [ ] 暗色模式 / 亮色模式（如支持跟随系统）

  性能测试

  - [ ] 扫码帧率稳定，不造成设备发热
  - [ ] 长时间扫码（> 5 分钟）摄像头不自动关闭
  - [ ] 连续扫码 10 个以上二维码内存不持续增长
  - [ ] 大图（> 10MB）解析不 ANR
